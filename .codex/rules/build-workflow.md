# build-workflow.md — 새 작업 생성 규칙

## 목표

새 작업은 작고 실행 가능해야 합니다. 처음부터 완성형 앱을 만들기보다, 1시간 안에 눈으로 확인할 수 있는 첫 결과물을 우선합니다.

## 생성 위치

여러 파일로 구성되는 작업은 `work/작업이름/`에 만듭니다.

폴더명은 가능하면 영문 소문자와 하이픈을 사용합니다.

```text
work/
└── my-first-automation/
    ├── README.md
    ├── src/
    ├── input/
    ├── output/
    └── notes/
```

## README 필수 내용

`README.md`는 다음 세션에 바로 이어서 작업할 수 있는 실행 계획서여야 합니다.

- 한 줄 목표
- 왜 만드는지
- 최종 결과물
- 입력
- 출력
- 필요한 도구
- 폴더 구조
- 만들 기능 체크리스트
- 첫 번째 작업
- Codex에게 다음에 요청할 말
- 확인 방법

작성 전 `.codex/references/work-readme-guide.md`를 읽고, 작성 후 `.codex/references/work-quality-checklist.md`로 점검합니다.

## 크기 조절

아이디어가 크면 최소 버전으로 줄입니다.

예:

- "생활 관리 앱" -> "CSV를 읽어 오늘 할 일 markdown 만들기"
- "유튜브 정리 도구" -> "자막 txt를 요약 노트로 바꾸기"
- "포트폴리오 사이트" -> "작업 1개를 소개하는 정적 HTML 만들기"

## 기본 기술 선택

초기 작업은 아래 순서로 단순한 도구를 우선합니다.

1. Markdown 정리
2. Python 로컬 스크립트
3. CSV/Excel 처리
4. HTML/CSS 정적 페이지
5. 웹 스크래핑
6. 외부 API 또는 배포

외부 API, 로그인 자동화, 결제/개인정보 처리, 배포는 기본안으로 잡지 않습니다.
