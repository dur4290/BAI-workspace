# security.md — 보안과 저장 규칙

## 기본 원칙

기본 작업에서는 실제 API 키를 만들거나 코드에 넣지 않습니다.

- 요약, 분류, 문장 생성은 Codex 세션에서 처리합니다.
- Python은 로컬 파일 처리, 텍스트 추출, CSV 정리, 폴더 정리부터 맡깁니다.
- 외부 API가 꼭 필요하면 비용, 키 관리, `.env` 사용법, Git 제외를 먼저 설명하고 확인받습니다.

## 금지

아래 값은 파일에 직접 쓰지 않습니다.

- `sk-ant-`
- `sk-proj-`
- `OPENAI_API_KEY`
- `ANTHROPIC_API_KEY`
- `GEMINI_API_KEY`
- `password =`
- `secret =`
- `token =`

아래 파일은 존재 여부와 경로만 언급하고 내용을 출력하지 않습니다.

- `.env`
- `.env.local`
- credentials 파일
- token/password/secret이 이름에 들어간 파일

## Git 저장 전 확인

`/save` 또는 Git 커밋 전에는 반드시 확인합니다.

1. `git status --short`
2. `git diff --stat`
3. 새 파일과 수정 파일에 민감정보 패턴이 없는지 확인
4. `.env`류 파일이 포함되어 있지 않은지 확인

문제가 있으면 커밋하지 말고 어떤 파일이 위험한지만 말합니다. 실제 비밀값은 출력하지 않습니다.

자동 보안 훅도 실제 값을 화면에 출력하지 않고, 위험 종류와 파일 경로만 보여줍니다.

## 위험 작업

아래 작업은 실행 전에 확인을 받습니다.

- 파일/폴더 삭제
- 대량 이동 또는 이름 변경
- Git push
- remote 변경
- 외부 서비스에 쓰기 작업
- 로그인 세션이 필요한 자동화

확인 요청에는 영향 범위, 되돌리는 방법, 검증 방법을 포함합니다.
