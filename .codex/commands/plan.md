# /plan — 작업 카드만 만들기

구현하지 않고, 막연한 아이디어를 작고 검증 가능한 작업 카드로만 정리합니다.

`/build`가 기본 진입점입니다. `/plan`은 사용자가 "계획만", "정리만", "아직 만들지는 말고"라고 했을 때 사용합니다.

## 사용법

```text
/plan 매일 확인하는 공지 페이지를 자동으로 정리하고 싶어
```

## 실행 순서

### 1단계: 아이디어 파악

인자가 있으면 그 내용으로 시작합니다. 인자가 없으면 어떤 것을 만들고 싶은지 묻습니다.

### 2단계: 필요한 것만 질문

질문은 한 번에 하나씩, 최대 4개까지만 묻습니다.

```text
Q1/4 — 이 작업은 어떤 입력을 사용하나요?
예: 폴더, 파일, 웹페이지, 메모, CSV
```

```text
Q2/4 — 결과를 어떤 형태로 받고 싶나요?
예: Markdown, Python 스크립트, HTML, CSV/Excel
```

```text
Q3/4 — 가장 귀찮거나 실수하기 쉬운 부분은 무엇인가요?
```

```text
Q4/4 — 성공했다고 보려면 무엇이 확인되면 되나요?
```

### 3단계: 작업 카드 저장

저장 경로:

```text
00-inbox/plans/YYYY-MM-DD-짧은-주제.md
```

작업 카드에는 아래 항목만 포함합니다.

```text
목표:
입력:
출력:
성공 기준:
오늘 만들 최소 버전:
나중에 개선할 것:
예상 작업 폴더:
첫 번째 작업:
```

### 4단계: 다음 선택지 제안

저장 후 이렇게 묻습니다.

```text
작업 카드를 저장했어요.
바로 `/build`로 첫 결과물까지 만들까요, 아니면 범위를 더 줄여볼까요?
```
