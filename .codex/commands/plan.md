# /plan — 아이디어를 실행 계획으로 정리

막연한 아이디어를 실제로 만들 수 있는 작고 검증 가능한 계획으로 줄입니다.

## 사용법

```text
/plan 매일 확인하는 공지 페이지를 자동으로 정리하고 싶어
```

## 실행 순서

### 1단계: 아이디어 파악

인자가 있으면 그 내용으로 시작합니다. 인자가 없으면 어떤 것을 만들고 싶은지 묻습니다.

### 2단계: 질문

질문은 하나씩 묻습니다.

```text
Q1/6 — 이 작업을 얼마나 자주 하나요?
```

```text
Q2/6 — 한 번 하는 데 얼마나 걸리나요?
```

```text
Q3/6 — 가장 귀찮거나 실수하기 쉬운 부분은 무엇인가요?
```

```text
Q4/6 — 필요한 정보나 데이터를 어디서 가져오나요?
```

```text
Q5/6 — 결과를 어떤 형태로 받고 싶나요?
```

```text
Q6/6 — 성공했다고 느끼는 기준은 무엇인가요?
```

### 3단계: 계획서 저장

저장 경로:

```text
notes/tasks/YYYY-MM-DD-짧은-주제.md
```

계획서에는 아래 항목을 포함합니다.

- 한 줄 요약
- 왜 만들고 싶은지
- 입력
- 출력
- 성공 기준
- 최소 버전
- 나중에 개선할 것
- 예상 폴더 구조
- 첫 번째 작업

### 4단계: 시작 제안

저장 후 이렇게 묻습니다.

```text
계획서를 저장했어요. 바로 `/build`로 작업 폴더를 만들까요?
```
