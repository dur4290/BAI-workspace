# /review — 저장 전 점검

작업을 공유하거나 저장하기 전에 완성도, 보안, 재실행 가능성을 확인합니다.

## 사용법

```text
/review work/my-automation
```

경로가 없으면 어떤 작업을 점검할지 묻습니다.

## 점검 순서

### 1단계: 파일 구조 확인

- `README.md`가 있는가
- `src/`, `input/`, `output/`, `notes/` 역할이 명확한가
- 불필요한 임시 파일이 섞여 있지 않은가

### 2단계: README 확인

- 입력과 출력이 명확한가
- 실행 방법이 충분한가
- 첫 번째 예시 입력이 있는가
- 결과 확인 방법이 있는가
- 다음 작업이 적혀 있는가

### 3단계: 코드 확인

`.codex/rules/coding-behavior.md`를 기준으로 확인합니다.

- 요청 범위를 넘는 기능이 없는가
- 불필요한 추상화가 없는가
- 경로가 개인 PC 절대경로로 박혀 있지 않은가
- 실행 실패 시 원인을 알 수 있는가

### 4단계: 보안 확인

`.codex/rules/security.md`를 기준으로 확인합니다.

- API 키나 비밀번호가 없는가
- `.env`가 포함되지 않았는가
- 외부 서비스 쓰기 작업이 있다면 확인 절차가 있는가

### 5단계: 결과 보고

```text
점검 결과:
- 반드시 고칠 것:
- 고치면 좋은 것:
- 바로 저장해도 되는 것:
- 다음 액션:
```
