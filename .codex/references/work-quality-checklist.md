# work-quality-checklist.md — 작업 생성 점검표

새 작업을 만든 뒤 아래 항목을 확인합니다.

## 구조

- [ ] `README.md`가 있다.
- [ ] `src/`, `input/`, `output/`, `notes/`가 있다.
- [ ] 개인 PC 절대경로가 들어 있지 않다.

## README

- [ ] 한 줄 목표가 있다.
- [ ] 입력과 출력이 명확하다.
- [ ] 첫 번째 작업이 작고 구체적이다.
- [ ] 확인 방법이 있다.
- [ ] Codex에게 다음에 요청할 말이 있다.

## 범위

- [ ] 첫 버전이 1시간 안에 확인 가능한 크기다.
- [ ] 요청받지 않은 기능이 들어가지 않았다.
- [ ] 불필요한 프레임워크를 쓰지 않았다.

## 보안

- [ ] `.env`가 포함되지 않았다.
- [ ] API key, token, password가 없다.
- [ ] 외부 서비스 쓰기 작업은 사용자 확인을 받도록 되어 있다.
