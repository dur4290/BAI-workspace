# work-quality-checklist.md — 작업 생성 점검표

새 작업을 만든 뒤 아래 항목을 확인합니다.

## 구조

- [ ] `README.md`가 있다.
- [ ] `src/`, `input/`, `output/`, `notes/`가 있다.
- [ ] 개인 PC 절대경로가 들어 있지 않다.

## README

- [ ] 작업 카드가 있다.
- [ ] 입력과 출력이 명확하다.
- [ ] 오늘 만들 최소 버전이 작다.
- [ ] 실행 방법이 있다.
- [ ] 확인 방법이 있다.
- [ ] Codex에게 다음에 요청할 말이 있다.

## 첫 결과물

- [ ] 빈 폴더와 README만 만들고 끝나지 않았다.
- [ ] 스크립트, 샘플 입력, 결과 파일 중 하나 이상이 있다.
- [ ] 실행 가능한 결과물이라면 실제로 한 번 실행했다.
- [ ] 실행 결과 또는 확인할 파일 경로가 남아 있다.

## 범위

- [ ] 첫 버전이 1시간 안에 확인 가능한 크기다.
- [ ] 요청받지 않은 기능이 들어가지 않았다.
- [ ] 불필요한 프레임워크를 쓰지 않았다.

## 보안

- [ ] `.env`가 포함되지 않았다.
- [ ] 실제 API 키, 토큰, 비밀번호가 없다.
- [ ] 외부 서비스 쓰기 작업은 사용자 확인을 받도록 되어 있다.

## 완료 전

- [ ] 공유하거나 제출하기 전 `.codex/references/project-finish-checklist.md`를 확인한다.
